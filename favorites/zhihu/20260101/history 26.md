# 如何让初音未来永远被记住？ niconiconi

**Author:** niconiconi  
**Bio:** 0.30000000000000004 年数值计算开发经验  
**Avatar:** ![](https://pic1.zhimg.com/v2-25d9d108639b1f93861f6d7fe26ebd66_l.jpg?source=0df5f383)  
**Author URL:** https://www.zhihu.com/people/5171caa9900e47b991a14d02b79bc715  
**Published:** 2024.12.23 02:21:49  
**Updated:** 2024.12.26 20:32:14  
**Question:** https://www.zhihu.com/question/566089141  
**Question Created:** 2022.11.11 23:18:22  
**Question Updated:** 2024.12.26 02:00:26

**计算机技术人，要抓紧时间利用 API 和爬虫存档元数据，留给人们的时间已经不多了！**

如果你仅仅希望保存角色与作品本身，那你什么都不用做。正如[@Lera](https://www.zhihu.com/people/1ccc40912a3b3dbdb6c8b27c212fbedb)所说：“不须要, 初音未来是会永远被记住的。初音未来是国际间最有名的合成人声乐器 [...]”，角色与作品肯定是会被写进历史教科书里的文化现象。**然而可怕的是，能向未来人们呈现该时代上下文的第一手来源很快就要消失殆尽了。**这就意味着，未来人们可能会无法理解“最初这是怎么流行起来的？为什么当时会这么流行？”的问题，后世的研究者看不到这些材料，就只能剩下泛泛而谈“岁月史书”了。以后历史老师或者教科书还会像分析“辛亥革命的历史意义”一样，将这个时代归纳为几个极为笼统、机械背诵的“划出的重点”，当时的时代面貌将无法被复原。

数据的缺失是两方面的：数据以及元数据。

- 数据即（原创与二次创作）作品本身——虽然不容乐观，总体问题不大。以中国大陆为例，大量早期作品已经因网站改版、博主删号、政策规定遭删除了，但是经典作品基本能得到比较广泛的存档，类似“NicoNico 殿堂曲三百首”这样的东西会长期存在于网盘、视频网站以及 P2P 网络上。
- 元数据的则泛指关于作品本身、并非作品的所有内容——目前情形已经十万火急。再以中国大陆为例：2012 年的初音未来的 A 站、B 站主页上能看到什么视频？一个热门视频播放量是在什么时刻开始暴增的？某个视频的推荐列表里又能看到什么视频？当时有哪些热门 UP 主？这个视频的发布者是谁？属于哪个创作团体？弹幕里都在说些什么？这一时期的社交媒体上有哪些梗？——这些信息的量正在锐减。而且不同于网页上的文字等表象，元数据基本无法被 archive.org 等现有的存档系统抓取，这些信息是动态从服务器获取的，只抓网页只会留下一堆不能用的脚本和死链接。
- 一个特殊的情况是：所有关于《魔法少女小圆》、《黑神话：悟空》、《东方兽王园》、《缘之空》的数据既不是数据也不是元数据，而是“圆数据”、“猿数据”、“园数据”和“缘数据”（感谢 @史蒂夫ZMT工作室 与 @日白残无Official 的补充）。

元数据将是未来读者理解本时代背景至关重要的信息，甚至比数据本身更为重要，然而这些东西几乎无人存档。未来我们面临的将是一个有数据，没有元数据与上下文世界——不，正是今天。B 站历史上许多现象级作品，弹幕现在都已经被清空了。如今看到的内容，都是后来再覆盖上去的新文字。

正如[@守望者](https://www.zhihu.com/people/25daeac0f99a82ebcb8f53ae522141a5)在《[为什么感觉现在的 bilibili 很没意思？](https://www.zhihu.com/question/626962666/answer/40148608741)》的回答中所写道：

> 假设你特别喜欢一部动漫或电影，想找一些关于这部电影的二创或解析视频来看。
> 
> 但是这部电影已经有些年头了，尽管它曾经在B站非常火爆，当时涌现出大量优秀的二次创作、剪辑和鬼畜段子等，但现在你可能一个也找不到。你的首页可能永远不会推荐这类内容，而且搜索栏搜索出来的结果也可能一言难尽。
> 
> 哪怕明明你敢兴趣的这些东西还存在B站的数据库中，但是给你的感觉就好像这部电影从来没有火过一样。推也推送不到，搜也未必搜的到。
> 
> 这样的例子在B站不胜枚举，B站曾经火过无数的电影，动画，电视剧，音乐什么的，但是只要时间一久他们就完全消失，就好像他们完全不曾存在一样，不留下一点痕迹，你有兴趣可以关注一下你的推荐栏，看看推荐的视频是不是都是最近三个月上传，最多半年，撑死一年，一年以上的视频就几乎不会推给你了。
> 
> 有人说这不正常吗？热度不是本来就一阵一阵的吗？过时了就不推了不是很正常吗？对于短视频来说确实是这样，但是长视频不是，因为长视频里面很多是有内容的，而优秀的内容是不会随着时间而过气的，是可以维持很长时间的价值的，一个好的长视频即使过个三五年甚至十年往上都会有人愿意看，更不要说一些专门喜欢早期歌曲明星和电影的观众了，B站的问题是这些过去的优秀长视频几乎推送不到喜欢他的观众那里。

即使在这些信息依然保留在网站数据库中的今日，元数据已经面临如此严峻的情况。当这些网站关闭时，后果将是不堪设想的——甚至比纸质媒体时代更为严重。在纸质时代，人们依然可以在图书馆、二手书店、私人藏书中找到当时的报纸、杂志、宣传册、笔记，尽可能复现古人的所见所闻（虽然很困难）。但如今，如果服务器一关，这些线索将荡然无存！

此外，你也应该已经意识到了，我说的不仅仅是“初音未来”或者“B 站”（两者仅是我最近发现的典型），而是任何网络媒介。例如微博、知乎、Twitter 也包括在内。

对于这个问题，有什么解决方法呢？

一、网络平台开发“快照”和“重放”功能（或至少是限定日期的“重现”功能），直接作为技术的一部分写入系统中。这一点，做得最好的是 Usenet（新闻组）——Usenet 是互联网有史以来最早的网络论坛之一。原因有两个，第一是因为只有文字，形式简单，没有今天音频、视频、文字、图片、个人主页、链接等过于嵌套的现象；第二是它具有标准协议、去中心化的架构。网络中每一台服务器完全平等，会互相把自己收到的贴子转发到整个网络的其他服务器的硬盘中，将其扩散到所有订阅者。任何一个参与 Usenet 的服务器（一个大学至少一台）都有大量用户，这些服务器订阅的新闻组足够多、历史存档足够长，最后其硬盘会把整个论坛的原始数据都抄下来。我至今依然可以用技术手段重现 1980 到 1990 年每一个时间点人们所能看到的内容，而且内容还包括完整的主机名、软件版本、发帖人、学术机构等（但可怕的是，即使是系统如此开放，也只有少数几台服务器留下了记录。至今这一时段的绝大多数历史资料来自多伦多大学动物学系的磁带，是 2000 年初从陈旧磁带中恢复的，其他学府的数据已荡然无存）。

![](https://pic1.zhimg.com/v2-9c0dc5940dd411762084606cda416629_r.jpg?source=2c26e567)

当然，那是互联网的田园牧歌时代了，现在肯定不会有这样天上掉馅饼的事情了。但是依然有一些系统支持历史重现。例如 Hacker News 首页支持“历史上的今天”功能，输入一个日期，就可以看到这一天的头条新闻并阅读评论，重现历史原貌。如果你是网络平台开发者，请考虑将数据、协议标准化，并在程序层面支持历史重放——然而现在说已经没用了，这个忠告留给下一代人吧。

二、网络平台开放原始数据库。如果网络平台本身并没有能力做到第一点，那么请至少在关站之前开放数据库，允许第三方开发者独立进行数据分析与二次开发——然而这是大概率不会发生的。

三、有技术能力的用户自行利用 API 与爬虫将重要元数据存档——用 API 可以获得、也有必要保存的有：视频本身（没有能力保存，这甚至不重要），视频的元数据（弹幕、评论、封面、播放量等），这些元数据本身所指向的元数据（例如说，如果视频元数据中有一个用户 ID，那么就有必要存在用户 ID 的元数据，如果这份元数据里还有头像、签名、发布过的视频等元数据，那么同样也存档）。还有一些内容，可能是 API 无法获取的——例如主页上推荐的视频、某个视频的推荐列表里又能看到什么视频等。这些内容是不固定的，可能需要有策略性的存几份典型，这些都可能是没有完美方法的复杂问题。此外，为了保证数据不会损坏，数据除了应该存入自己的数据库，还应该保留服务器返回的原始内容。所有存档产生的数据应使用多种媒介保存，并随着技术更新迁移到新媒介，老人退出时交给新人。

四、全部存档所有元数据可能相当困难，而且还会遇到法律挑战（如隐私问题）。此时可以开发个人元数据存档：开发一个机器人，机器人代替你登录账号，将你每天（以及曾经）在网站上的所见所闻（如时间线）记录下来并保存到本地，技术手段与（三）相同，只不过范围局限于自己接触到的那部分，而且不公开，仅保存在自己硬盘中。这样至少解决了某条消息丢失，日后自己后悔没能保存的的问题。

第三、四条路线是最有必要，也是最困难的路线。遗憾的是，我个人的时间、精力、身体状况，是无法开发这一浩大的项目了。我在这里提前向后人谢罪了！如果屏幕前的你有能力，这个时代的大业就靠你了。目前这一时代文化正在极速离我们而去，随着热度越来越低，以后要找到历史遗迹也越发困难；同时随着大型语言模型逐步推开，目前网络正走向全面封闭化、反爬虫化。看看 Twitter 和 Reddit，短短几年前什么信息都能用 API 检索，紧接着发生巨变，现在都无法查询了。公开的元数据已经越来越少了。时间窗口正在关闭。

如果不是现在，还能是何时？如果不是你，还能是谁？

这居然是个新提问。2025了我寻思已经是个二次元＝年轻人最新最潮的时尚单品的时代了，哪还有这种误解。现在你说自己是二次元，老一辈人眼里怕不是得：哦，谷子！哦，新风口！哦，这个玩意赚钱！

话虽如此，这个提问本身所说的现象。倒也并非完全不存在，倒也不一定是误解，关键在于究竟什么才是“现实”。所以就借此来谈谈当下宅宅的们所处的现实吧。

## 当下阿宅们所生存的现实

一般大众的直觉会认为，线下就是现实，互联网或者漫画书就不是现实。事实上，早在上个世纪 ，类似的批评和看法就已经存在了。

刘宝瑞有一段相声《书迷打砂锅》，大概是在五十年代表演的（我没有查证，但大概是这个时段左右）。其中前半段首先讲了七侠五义这些武侠小说都是编的（虚构的）而不是真的（现实的），然后讲了一个沉迷于听评书的清朝年轻人分不清现实和虚构，沉浸在评书的世界里，因为听《跨海征东》，听说书人说到唐王被困淤泥河就且听下回分解了，而开始幻想自己要去救驾，结果闹得全家不得安宁。

在这个相声里，很显然这个年轻人“沉迷虚拟世界”“逃避现实”的。相声本身毫无疑问对这个年轻人持有批判的态度。在这一时期，人们似乎认为现实与虚拟的区分似乎是不言自明的。

然而，事实上真的如此嘛？现实就是很现实，虚构就是很虚构？恐怕未必。

**我们不可能不借助任何媒介来认识现实**。而通过不同的媒介不同的角度观察，同一个事物呈现出来的样貌可能大不相同。我是个近视眼，摘下眼镜看到的事物都是模糊的，戴上眼镜才能够看清晰。那么到底模糊的是现实还是清晰的是现实呢？如果清晰的是现实，那么为什么哈哈镜就不是呢？

这并不是说现实不存在了，而是说现实的定义，是随着时代尤其是媒介环境的变化而变化的。以古登堡的铅活字印刷术的发明为代表，书籍变得廉价，使得报纸开始成为人类认识现实的重要媒介；以达盖尔银板摄影术为起点，彻底改变人们对绘画的认知。

我们甚至可以说，恰恰是这些技术的发明，才使得虚构和现实的区分开始变得清晰。在古代，神话传说、虚构故事，和人们的认知中未必有那么明显的区分。所以才会“敬鬼神而远之”。

那么随着报纸、广播、电视、互联网甚至移动互联网的发明，我们对于现实的认知不也在被改变着么？千里之外发生的事情被写成报道、甚至被拍摄成抖音，我们对于千里之外的了解在当下恐怕远远超过了对于身边事的了解。在这种情况下，到底哪一个对我们来说更加现实一些呢？

扯得有点远了，让我们回到宅宅这里。宅宅们所生活的环境，是一个被各类文化创作物所包围的环境。因此对于宅宅来说，这就是现实。不是说宅宅们会把哪个哪个动画片世界观当真，而是说消费这样的产品对于宅宅们而言是理所当然的。

在当下，移动互联网如此发达，社交软件、视频软件、游戏、线上购物已经全面重新塑造了我们所要面对的现实，再来说什么逃避现实已经完完全全的不合时宜了，**网络就是现实**。

以前，老师家长们会对我们说：学校是个小社会。意思是说校园是对于社会的模拟和预演。而现在，我们这些生活在互联网上的z世代们会反过来说出这样的话：**社会是个大手游。**

![](https://picx.zhimg.com/v2-8c53040ed2cf9f245fbe308fcdd540a3_r.jpg?source=2c26e567)

代表着虚拟世界的互联网以及其一系列衍生，重构了当代年轻人认识现实的方式，并且再生产着现实。不只是年轻人，父母甚至老人也都在刷抖音。互联网已经是现代文明社会基础设施建设了。在这个意义上，虚拟世界已经不存在了，而已经彻底成为了现实。也因此，曾经（互联网方兴未艾时期）的宅宅们关于虚拟世界与现实世界之间的张力也消失了。

## 00年代前后阿宅们的样貌——《刀剑神域》和《记录的地平线》

在00年代的宅物中，有一些非常能够体现虚构与现实的差异问题的作品，比如说《刀剑神域》。如果要一个个细节分析根本讲不完，这里就举一个例子吧。以下来自第十七卷中莉兹贝特的台词：

> 「我和她正在只聚集了你说的生还者的学校上学。根本没有选择的余地……在那个学校里，每周都要做一次心理咨询……被质问各种『会不会失去现实感啊』『会不会想伤害他人啊』这样无聊的问题。甚至还出现了被强制进行药物治疗的人。**在政府眼中，我们都是监视对象的犯罪者预备军呢。**」
> 「但是……其实，受到这种待遇的并不只是生还者学校的学生们。VRMMO玩家都或多或少被这样看待着。**在他们眼中，网游者是社会的寄生虫，是不交税金和养老金的现实逃避者……甚至还出现了应当恢复征兵制度，强制我们进行社会服务的论调！**」
> **「但是，我知道！我也相信！现实就在这里！！」**
> 「**这个世界，以及连接着这里的无数虚拟世界，绝不是逃避来的虚构之处！这里有真正的生活，真正的朋友，真正的欢笑、泪水、邂逅以及离别……**这里就是『现实』啊！！大家也是这样对吧！？正因为我们相信这个世界才是现实，所以才会如此努力的，对吧！？**然而，如果认为这只是游戏……只是伪造的虚像就舍弃的话，那我们的『真实』又在哪里……！！**」

正是因为在这一时期互联网的世界和线下的世界还存在着根本性的差异。所以才有想象把虚拟世界当做寄托真情实感和超越线下社会闭塞感的可能。

上面引用的是来自日本的青春文艺作品中体现出的价值观。而在中国，在这个层面上，青少年们面对的问题是类似的。而中国这一时期，也有着类似的表达：网瘾战争。

[【看你妹之网瘾战争 4K超清纪念版-哔哩哔哩】](https://link.zhihu.com/?target=https%3A//b23.tv/FuCHBjQ)

不只是网络游戏玩家，在动画爱好者那里，曾经不也有过“天下漫友是一家”等类似的想象么。

当然，如果这样的话，很容易把这一时代下下玩家们或者说阿宅们理解成某种“选择相信”的“二次元存在主义”。例如王玉玊博士对于《记录的地平线》第二季第10集威廉的解读：

## 总结

这里大概以游戏宅群体为案例讲了一下游戏宅所谓的现实与虚构到底是怎么回事。

00年代的游戏宅们所追求的，正如邓剑在《MMORPG网络游戏批判》中所说：

> 玩家不只是在游戏世界里机械化地按照游戏程序的目标设定升级、打怪，更是**在“游戏社会”里自由交往、追逐梦想，寻求一种与现实生活中的琐碎、压抑、沉闷等负面感受拉开距离的游戏化生存的可能。**当时（2000年代初）正值中国的互联网建设起步，人们普遍期待彼此之间的闭塞感可以借助数字信息技术的力量打通，从而为日常生活赋予新的可能。在这样的社会现实配合下，玩家们逐渐体认到MMORPG网络游戏带来的广泛且真实的共同感，并据此具体地想象“地球村”（马歇尔·麦克卢汉，1964/2011：75）的基本样态——他们在互联网化的游戏世界中交易、交友、结婚、结社，践行一种自由、自在与自为的21世纪乌托邦想象，甚至迫切地将游戏世界中的交往关系也延伸至现实社会里。MMORPG网络游戏逐渐在他们的精神生活中占据十分重要的位置（甚至会被现实社会解读为游戏成瘾），一种关于游戏乌托邦的想象在2000年代初诞生了。

而随着互联网技术逐渐深入到整个社会当中，这种乌托邦想象也在消失。当今的人们依赖于互联网生存，所以面对的状况不再是所谓的在虚拟世界里逃避现实，而是找不到一个可以逃避的世界了。对于生活在互联网中的人们，互联网伴生的世界就是现实的一部分，乌托邦的想象已经不是网游或者说互联网可以承载的了。

实际上，现实和虚拟的问题，或者说宅宅们，日本或者中国，乃至全球的青年文化的问题是非常复杂的，和技术环境、文化环境、政治环境、历史环境都息息相关，我这里说的也只是冰山一角。

现在持续运营网络游戏的新作品越来越难以生存，无论哪一部作品都会很快燃烧殆尽。严格来说，大部分企划在到达正式发布之前就已经消亡。这种趋势在2020年后愈发明显，从data.ai（原App Annie）的数据来看，移动游戏市场中全新IP能够到达市场高位的案例每年都在减少，整个市场的新陈代谢几乎停止。这种情况并不是单纯的“竞争激烈”导致的，很大原因是在手游市场，模仿过去的成功经验的开发设计仍然有很多，而这些设计现在已经无法简单地生效了。雪上加霜的是，现在开发游戏的成本规模每年也在不必要地扩大，这些因素导致了整个环境结构性的扭曲。那么，这样的扭曲究竟是何时开始？矛盾在哪里？以及是否有突破口存在？关于这些问题，我想结合我的经验进行思考。




## 第1章：改变了用户评价标准的“原神冲击”

2020年9月，《原神》隆重发布。miHoYo（现 HoYoverse）推出的这款作品，似乎给日本游戏业界和游戏用户带来了冲击。高标准的画质、多平台同步发行、以及更重要的是符合日本市场喜好的赛璐璐动画风格的画风、出色的运镜和表演，每一点都打破了移动游戏的常规。开发成本与广告成本都突破了数百亿日元规模，极高的内容细节的完成度，这些都使得玩家对游戏的预期发生剧烈的改变。

**原神诞生带来的错觉与日本因此踏入的歧途**

随着原神的诞生，在移动游戏的开发领域，在我看来整个行业开始浮现出几种执着的观念：

“是不是游戏不搞大制作就无法成功？”

“维持现状真的好吗？”

“制作游戏是不是需要突破性的内容和新鲜感？”

特别是，在拥有决策权的人群中这种认知越来越强烈。

然而，日本国内的手游开发的预算原本就无法与中国大陆游戏相比。模仿开发体制和制作方式只会加速死亡的可能性更高，我认为应该将思考方式转移到体验、趣味性和重新审视商业模式上。从目前独立游戏产业的热度和之前超休闲游戏流行的趋势来看，我个人的反思有很多。

另外无论日本国内外都有追随原神的Genshin-like作品，但没有任何企业能够复制原神的成功，反而筑起了一座接连失败的高山。结果而言，这使得游戏制作规模越来越大，开发成本一路膨胀，但只留下“不如原神”的一地鸡毛。

在2020年以前，每月 1 亿日元左右流水的项目仍然有微小的生存可能性。但原神之后，要达到收益的水准大幅提升，公司变得无法等着你一边运营一边改进游戏，导致项目在早期就被终止的情况增多。这对玩家来说，会留在市场的好游戏变多是一件好事。但对开发者来说，做游戏的难度确实剧烈提高，许多商业上的失败不断持续。

## 第2章：是什么导致了商业上的大失败

我也曾参与开发、发行过许多运营型网络游戏，但失败案例也相当多，如果被问及为何商业上会失败，确实有不少值得深思的环节。例如，企划不成熟、缺乏趣味性、未能制作出符合预算的游戏体验、竞争对手过于强大、产品完成度不足、出现致命性 Bug、公司跑路、侵犯了大IP版权等等，我几乎踩过所有常见的陷阱。而我可以断言的是，在这些问题中最常见的就是，“作为长期运营游戏的玩法不成立”、“无法与习惯性的运营模式做出差异化”、“没能打开新市场”。说到底，为什么花了好多年也没能把游戏做出来？这其实是因为在开发过程中，有着很多资深制作人也难以察觉的结构性问题与陷阱。

**想要商业成功所需要的四个要素**

要让移动端运营型游戏成功，我认为不仅需要一位天才，至少还需要同时具备以下四种不同的技能与品质。

**体验设计能力**——能够长期让玩家产生“再来一次”想法的游戏本身的设计能力与开发技术

**运营设计能力**——能够轻松、快速、通过反复测试进行轨道修正的能力

**收益设计能力**——能够在不破坏用户体验的情况下，设计长期盈利和游戏平衡的能力

**经营判断能力**——包括广告的操作在内，能够判断宣发的打入、停止、继续，并设计撤退线的能力

**为什么需要这四点**

长期运营游戏之所以难以生存，因为在游戏正式版开发完成的那一刻才算站在起跑线。买断制游戏在作品打磨完成并正式发布后便告一段落，然而运营型游戏在发布之后才进入“设计的第二章”，需要持续不断地进行平衡调整、内容更新、收益管理和决策判断。换句话说，必须以完成的作品为基础，设计出"不断游玩、持续进化"的结构。

**与制作买断制游戏不同的难点**

想象你要持续运营一款买断制RPG大作，你是否能够将数值与内容无缝切割，为其设计长期的更新路线图？而且，要维持用户数量的同时，还会时刻因为部分玩家在你预想之外的发现，造成游戏社区氛围与角色最优配队不断改变。在这个情况下，想让游戏如你所想运营数年，仅靠游戏设计师个人的能力根本不可能实现。在从前，因为哪个作品都会有很多人玩，这样即使项目有所缺陷，获得的收益也足够掩盖过去。如今这些红利已经消失，每个游戏都面临着会被与其他游戏作比较的严峻环境。因此想要成功需要靠组织来创造“可复现的天才”，这也是运营性游戏开发的最大难关。

原本游戏设计师应该是专注于"趣味性"的职业。但在运营型游戏时代，他们必须同时持有"可持续性·盈利性·商业判断"的视角。换句话说，如今游戏策划必须从玩家心理和经营者的双重视角进行设计。因此，上述的4个要素如果不能整合在一起的团队就无法成功，无论缺哪一点都会导致项目的崩溃，这个认知是绝对不能忽略的。

## 第3章：长期运营游戏的结构性困难

虽然刚才讲了让运营型游戏商业成功需要的四个要素，但首先要认识到，运营型游戏的设计本身其难度远超买断型游戏，这是一个现实。若没有认识这一点就进入开发，无论多么优秀的团队都会失败。

**“不能让游戏结局”这一诅咒**

买断制游戏有着明确的结局，玩家打完结局满足地封盘，开发者只需设计好“直至结局的体验”即可。无论是 10 小时还是 100 小时，正是因为有结局的存在，设计才成为可能。但是运营性游戏没有尽头，应该说是“不能结束”，因为一旦结束，游戏就得停服了。因此需要不令玩家厌倦得让他们玩上数年，这有多么困难。一款买断制的史诗级 RPG，即使是一款能玩 100 小时的杰作，通关后选择「再玩一次」的人也很少。许多玩家在看到结局后会感到满足而离开。但是对于运营型游戏，为了不让玩家离开，就必须每个月持续提供新的内容。这就像连载漫画一样，要持续画上好几年，就要对抗创意枯竭、模式化、读者流失等问题。更困难的是，玩家消费内容的速度远大于供应速度。开发团队花了一个月制作的活动，核心玩家一天就能全部玩完。然后玩家就会要求你更高的产能，但是提高产能意味着开发费用和人力成本急剧增加。运营团队疲惫不堪，内容质量也随之下降，想要解决这种不平衡非常困难。

**手游特有的限制**

**操作的限制：**许多运营型游戏都是为移动设备设计的，但是这其中会遇到很多限制。在主机游戏上，手柄能够有10个以上的按键，这使得玩家可以做出复杂的操作与判断。但是在移动设备上只能依靠触摸屏。想在屏幕上配置虚拟按钮，要注意到手指会遮挡屏幕画面，这使得如果要让玩家做出复杂的操作，同时也会给玩家带来体验上的压力。结果，手机游戏就是会被限制在“简单的操作方式”上，于是玩法的设计也会被极大地限制。因为玩家并不期望手游需要复杂的操作才能玩，这也让创造新的玩法体验变得困难。在单机游戏中，新的操作体验会带来革新。《塞尔达传说：荒野之息》的物理运算、《死亡搁浅》的移动体验、《只狼》的弹刀系统——这些都以复杂操作为前提。在移动端游戏这样的玩法革新难以产生，模仿既有成功经验的玩法被大量生产。

**画面的限制：**同时手机的画面更小，复杂的UI会使得可识别性下降，导致误触率更高，无法像单机游戏那样“运用整个游戏来表示信息”，因此必须削减画面的信息量令其简洁易读。但过于简单又会失去游戏的深度和战略性，想取得平衡非常困难。

**时间的限制：**移动端游戏是以“上班路上”“小休时间”“睡前5分钟”这种碎片时间段为目标而设计的。一次要玩30分钟以上不能停下来的设计在移动端游戏中需要极力避免的，玩家寻求的是“稍微玩一下”而已。但是，在这种碎片时间想提供给他们满足感也很困难，深刻的体验、成就感、故事——这些都需要时间。结果就是，因为游戏设计更偏向于“短平爽”，也让移动端游戏的体验变得趋同。

**以长期运营为前提的设计难度**

运营型游戏需要在发布时满足“3 年后也能玩的设计”要求。但是，预测 3 年后的市场环境是不可能的。

**数值膨胀：**运营时间越长，角色和道具的数值就会越高，新角色如果不能够比现有角色更强，那就不会有人抽卡。但是，膨胀过大会让新玩家感觉到不公平，新玩家会感到追不上整体强度，从而离开游戏。

**预留设计空间：**如果开服版本就是一款“完整的游戏”，那就会失去追加内容的空间。等级上限、装备和技能的强度种类，在开发时就要留出扩展的空间，但要是留太多，又会给人游戏完成度低的感觉。

**T0的变化：**当游戏社区形成，“T0”也随之产生。哪个配队最强、哪个角色弱——玩家间的交流就会逐渐形成游戏的最优解。但是如果最优解长期固定，会让玩家对游戏感到乏味，即使推出新角色，不能够参与进最优配队的话也不会有多少销量。开发者为了打破最优配队的地位就会不断进行平衡调整，但是如果做得过火，就会遭到“策划削弱老角色”的批评引发节奏。策划几年间都需要持续这样的走钢丝游戏。

**创造“领先1.5步”优势的难度**

市场已经不是“从0到1”而是“从1到1.5”的时代了。但是想创造1.5步的领先是最难的。管理层倾向于规避风险。他们会做出“这个企划和XX类似，所以是安全的”判断。然而，模仿现有作品会被玩家看穿指责其为“〇〇的抄袭”，最终会被埋没。但过于创新的项目难以获得高层理解导致无法通过预算。并且玩家也既追求“新鲜感”同时需要“熟悉的体验”，应对这种矛盾需要的就是“1.5步的领先”。但1.5步究竟是多远？没有人有标准答案。结果就是大部分企划都停留在0.3步被淘汰，或者迈出了3步远却没有人能理解。

**轻视游戏设计难度的业界**

尽管存在如此多的困难，整个业界却轻视游戏设计的难度。"只要做出好玩的游戏就能卖出去"——这种简单的想法导致了许多失败。什么是好玩？如何设计出不会让人感到厌倦的游戏？在移动设备的限制下，如何实现创新？对于这些问题，没有人能给出答案。正因为如此，在接下来的章节中，我们需要阐述的“结构改革”就变得至关重要。正视游戏设计的难度，如果不改变组织、流程和决策权，运营型游戏将无法生存。




## 第4章：在游戏核心循环逻辑完成之前就开始行动

无论什么种类的游戏，要确定游戏的“乐趣的核心(Core Loop)”都需要时间，但是开发现场等不了。理由很简单，因为公司是不能停的。游戏的开发是多个职能同时阶梯式运行，任何一个职能的停摆都会导致整体的停摆；外包和员工薪资都要按月结算，项目停止成本很高；管理层追求项目的进度，会将实验视为停摆。结果，整个项目会陷入“虽然根本不好玩但是还是得做下去”的状态。

**“无法说出真话的结构”杀死了游戏的玩法**

许多项目都会有很多外部合作方构成。但是，许多外部合作者即使看出来“这个设计很失败”也说不出口，因为如果开发停滞了他们也无法获得收入了。（*当然，并非所有事情都适合说出口）如果说“想要重做”会被当作“延误工期”，评价也会降低。有时还会签订开发中止就不会结款的合同，在这种情况下，对于他们来说，停止开发游戏是没有好处的。

遇上会说“你要是这么想那就别干了！”这种暴躁的制作人，或者开发者也因为考虑到经济问题只能别人说什么他做什么。在这样的环境中，大家都不敢说出真心话，就这样把项目推进到了预定发布日。

**没有时间打磨玩法核心循环**

实际上，优秀的循环并非是照抄理论，而是从"手感"中发现的。最重要的是"游戏是否好玩"，但往往因为项目档期的限制，游戏就在没有进行打磨或路线调整的情况下强行推进。这是我自己也多次经历过的，如果玩法能在期限前打磨好那谁都不会辛苦了。但既然会下预算那就要有档期，即使可以稍微延期，但很少有项目能够停下来重做测试的时间。结果在开发现场，UI与剧本也在同时推进，试错的时间也被压缩，最后“漂亮的未完成品”就被生产了出来。

**将踩刹车的勇气化为制度的公司**

另一方面，成功率高的公司究竟有何不同？例如，听说 Cygames 严格执行“在玩法确定之后再开始正式生产”的原则。核心引擎完成前不制作车体，能够严格执行这种留有余地的公司就很强。然而中小企业会面临现实困境，一旦停止就会资金耗尽。这里就产生了“结构性差距”。许多企业预算和档期都十分严格，玩法设计团队没有留有余地的时间，这或许正是导致商业失败的重要原因之一。无论如何，在企划、开发、经营、外包——所有环节都处于"无法停止"状态的项目，在当今时代，无论怎样都极有可能失败。我几乎不知道有什么成功案例。这种结构性缺陷能否得到解决，可能成为运营型游戏能否生存的分水岭。

## 第5章：无法刹车的团队结构

玩法核心循环没有完成就不得不发的背景中，还有很大原因在于，团队的组织决定了无法停止。在开发现场，项目进度本身被视为“考核标准”，而停止则被视为“不合格”。然而，在游戏开发中，项目的进度并不总是等于项目的成长。

例如，假设某月企划团队意识到“这样不够有趣”，但 UI 团队、声音、剧本、市场营销已经启动。要停止就需要将所有工作回滚，没有人愿意承担责任，于是就会产生“总之先做着事后再改”的判断。但是，游戏发布后变得更有趣的例子少之又少。反而一旦用户在初期就认为你无聊，那他们再也不会回来，市场份额也不会回来，最终没有踩刹车的项目，会在后期遭遇数倍于开发时的损失。

**没有拥有“刹车权限”的人**

在许多项目中，其实没有一个真正的决策者。“谁来喊停一下？”这个问题没有人能够回答。对项目总经理来说，因为负有销量的责任所以很难叫停；对制作人来说，因为负有领导开发团队的责任所以很难叫停；对公司管理层来说，因为不了解开发现场的细节所以很难叫停。

结果而言，因为没人能够踩刹车所以项目会持续推进，"正在进行=就是好事"的错觉已经渗透到整个组织。这才是劣质作品大量产生的真正原因。

**有“进度会议”却没有“决策会议”**

在实际开发中，进度汇报会议会频繁地进行，但是，“是否应当继续项目”的会议却不存在。“开发进度到80%了”、“距离公测还有3个月”这样的报告接连不断，但是却没有能说出“虽然已经完成80%但是这游戏其实根本不好玩”这种话的气氛，谁也不会来踩刹车。所以项目不会停止，然后项目要上线了。

## 第6章：运营成本的重压正在扼杀创造力

长期运营游戏的现实是残酷的，用户数量基本是会一直减少。无视这个现实而设计的运营模式必定会失败。

**大制作=短命的悖论**

许多运营团队会将大制作误认为成长，但是制作规模越大的游戏运营成本也越高，这使得更新速度会变慢，游戏寿命变短。例如对于大型游戏，考虑到每月的服务器成本和人力成本，每个月的制作成本会达到数千万日元，按年度算就会形成每年数亿日元以上的固定成本，这不是续命策略而是变成了自毁策略。大制作能够满足用户暂时的满意度，但是会让团队疲于长期维持更新节奏。

**画面和趣味性并不成正比**

《吸血鬼幸存者》是一款由单一名开发者制作的像素画游戏。但它凭借其"玩法循环的舒适感"获得了全世界玩家的反响。相比之下，花费数十亿日元的大作在数月内停止服务的例子并不罕见。游戏的趣味不在于制作豪华，而在于“重复获得快感的结构”，没有重视这一点的游戏，投入再多的预算也是石沉大海。

问题在于，即使开发人员理解了这一点也无法停止。开发一旦停止，外包和合同工的合同会结束，正式员工的绩效会下降，没有人愿意接受损失。因此即使知道不好玩，也没人会说出口，没有人踩刹车，于是又会生产出一个徒有其表的平庸作品。

## 第7章：市场正从“0→1”时代迈向“1→1.5”时代

手游的黎明期是“从0到1”的时代。只要创造出新的体验，市场就会给你反馈。在当时，抽卡、联机、体力系统、放置玩法，这些都是“新鲜感”本身。但今非昔比，玩家的体验期待值已经饱和。现在需要的不是新事物，而是“领先1.5 步的体验”。许多模仿XX的游戏，只是模仿了表面，却没有复制出玩法核心的体验。只在“探索的乐趣”、“节奏感”、“操作感”这些方面前进了0.3步，于是泯然众人。

**市场已成熟，模仿成为风险**

“模仿热门作品就不会出错”的时代已经结束了。现在情况相反，如果模仿热门产品，那么就会和其他100家公司站在一个起跑线上，“相似”本身就会成为风险。当FGO流行的时候，业内开发了不少模仿FGO的游戏，但是大多数都在做完之前就被砍了。

根据data.ai的数据，每月都有数百款新作发布。但只要看看 App Store 或 Google Play 的前 100 名，就会发现其中大部分都是现存的游戏。新游戏想要进入排行榜，只有大型企业的大型 IP 游戏或者投入巨额广告费的游戏才能实现。中小企业的新作几乎没有进入的机会。也就是说，游戏市场已经陷入“数量过多导致无法做出差异化”的状态，无论制作多么有趣的游戏，能够让玩家发现本身就很困难了。

但是即使游戏能提供“1.5步领先的体验”就能取胜吗？其实答案也是错的，因为玩家根本就没精力再玩新的游戏了。

## 第8章：“时间与金钱的困境”——沉没成本阻碍了新玩家的产生

对于长期运营游戏，缺乏新玩家并非单纯是制作组的问题，在玩家群体中，也有着结构性的壁垒存在。那就是玩家在已投入游戏中的沉没成本。

**金钱与时间的双重牢笼**

玩家已经在他们正在玩的游戏中投入了大量的成本，不仅是金钱，还有数年的游戏时间和养成数据，以及好友、社区中的人际关系。这个在一个游戏中积累的东西越大，玩家转移到另一个游戏的心理门槛就越高。“从头开始”、“过去氪的金就白氪了”——这类的情绪会阻止玩家去玩别的新游戏。这是行为经济学中所谓的“沉默成本效应”。人们不愿让已经投入的东西白白浪费，而运营型游戏正是巧妙地利用了这种心理进行设计。

**运营型游戏的“围栏设计”**

围绕玩家心理，运营型游戏还故意构建了难以退出的结构。每日任务、限时活动、登陆奖励、月卡季票、好友系统等，如果不参与就会让玩家感觉到损失。这些机制维持了玩家的留存，同时也剥夺了玩家去转移到新游戏的空间。玩家在物理上可以同时深入游玩的游戏是有限的，只要被现有游戏占据了时间，新游戏就失去了插足进入的空间。

同时玩家玩游戏的时间也没有增长，除开工作，聊天、长视频、短视频，网络的娱乐选项也不断增多。大多数玩家已经在同时玩着1到2款运营型游戏，算上日常与活动，那么平均每天都有1到2小时时间投入在既存游戏中。而新游戏必须要夺走既存游戏“已经占用的时间”。但这并不容易，因为玩家更偏向回避损失，不想抛弃已经积累的数据的心理在起作用。

**“让用户离开”的难度**

新游戏要想成功，必须达成以下一项。

**1.**让玩家完全抛弃旧游戏来玩新游戏——极其困难

**2.**让玩家同时玩旧游戏与新游戏——这就形成了游戏之间争夺时间的消耗战

两者都是荆棘之路。特别是在原神之后，业界都开始倾向于大制作，因此游戏对玩家游戏时间的需求变得更加强烈。这使得玩家在物理上无法同时玩3或4款游戏。

结果，新游戏就必须要“让玩家离开还在玩的游戏”了。

**中小企业的绝望**

如果是大型企业，凭借大规模的营销和 IP 的力量，可以“强行制造热度，让一般玩家暂时从现有游戏中脱离出来”。但是中小企业没有这种能力。即使花费广告费诱导玩家下载，玩家之后也会选择“回到现有游戏”。游戏首次启动后的流失率异常高，正是由于这种结构所致。打完教学关，玩家就为了完成现有游戏的每日任务而离开，然后便再也不会回来。所以在探讨玩法乐趣之前，如果不能提出“为何要放弃现有游戏来玩新游戏的理由”，新游戏就无法生存。

**数据显示的残酷现实**

据一项调查，大约70%的移动游戏玩家回答称“因为已经有在玩的游戏，所以不打算开始玩新游戏”。此外，即使安装了新游戏，80%以上的玩家会在一周内离开。其中许多原因是“没时间”和“已经玩着其他游戏”。也就是说，玩家并不寻求新游戏，他们在现有游戏中已经获得满足。制作者无论做出多么好玩的游戏，玩家的时间和金钱都会被现有游戏固定化。不突破这道墙，新游戏就无法生存。

要突破这种结构，可以考虑以下策略。

**超短游戏时间设计：**瞄准与现有游戏无竞争的间隙时间，单次游戏可在5分钟内完成，并且没有每日任务的设计。

**完全买断制：**不会产生沉没成本，提供“有终点的体验”。回归买断制避免与现存游戏竞争。

**与现存游戏的差异化：**避免游戏类型相同，提供完全不同的体验，例如重视故事、创作者工具、社会实验性元素等。

**抛弃社区：**游戏可以单人完成，具有“即使不玩了也不会受到影响和造成影响”，完全排除好友系统和社区，避免产生沉默成本。

但是这些会颠覆“运营型游戏的常识”。沉没成本壁垒，无论开发方多么努力，终究存在于玩家一方，因此难以轻易打破。无视这一现实，还相信“只要做出好玩的游戏就能获胜”，这已经近乎幻想了。

## 第9章：隐形成本吞噬利润

长期运营游戏的收益结构还存在另外一个问题，就是隐性成本。游戏的开发费用和广告费用虽然容易可视化，但实际上大部分销售额都消失在“看不见的成本”中。中小企业无论销售额有多高都难以盈利，正是这个结构造成的。苹果和安卓商店会征收1成到3成的平台税，这个高额的费用对中小企业来说是一个致命的负担。此外，还有支付结算手续费、服务器费用、客服费用等。将这些费用合计起来，35%至40%的销售额会以“平台成本”的形式消失。

**广告费用飙升——CPI/CPA的地狱**

其次，就是游戏的获客成本、在移动游戏市场中，几乎无法期待有机流量（用户自然自发的下载），想要获取新用户只能投放广告。但是，广告费用也在逐年飙升。

**CPI（Cost Per Install）：**每次安装成本，每个安装的单位成本。目前的平均值为300至800日元。

**CPA（Cost Per Action）：**每次行动成本，获取一个付费用户所需的成本。平均为5000至15000日元。

也就是说，厂商为了获得一个付费用户，需要花费超过1万日元的广告费。如果这个用户在其游戏周期内没有充值超过1万日元，厂商就会亏损。但是，很多玩家不会充值，充值率平均在5%以下。此外，会充值的用户的平均LTV（生命周期价值）大约也在1万日元左右。一旦依赖上这个结构，游戏就会变成“获客越多亏损越多”。

**IP与开发的固定成本**

此外，如果是知名IP作品，还会产生IP的授权费。根据合同形式不同，通常情况下会是“销售额的x%”或者“最低保证金+销售额的x%”的形式。例如，如果合同规定支付10%的销售额作为IP授权费，那么月销售额1亿日元中1000万日元会作为IP成本消失。算上平台税30%，此时你就已经失去了40%的销售额。

并且，游戏运营本身也有固定成本。内容更新、运营的人力成本，服务器费用，客服费用，活动组织成本等，相加起来就会是巨大的金额。

**无法盈利的成本结构**

将以上内容整理一下，假设一款月流水1亿日元的游戏的收入结构就会如以下所示：

**流水：**1亿日元

**平台税（30%）：**-3000万

**广告费：**-2000万

**IP授权费（10%）：**-1000万

**人力、运营成本：**-3000万

**盈余：**1000万

也就是说营业利润仅为10%，而且，这还是收回开发成本之前的数字。如果开发成本高达2亿日元，那么需要运营20个月才能收回成本，但是许多游戏根本撑不到20个月。即使月流水能够到达1亿日元，利润也所剩无几，这就是运营型游戏的现实。

如果是大型企业，可以同时运营多款游戏，用成功作品的盈利填补亏损游戏，但是中小企业没有这样的实力。一部作品失败，公司就岌岌可危，两部作品接连失败，公司就会破产。平台费用和广告费都与企业规模无关，实行统一标准。也就是说，中小企业在结构上处于不利地位。

**是否有突破口？**

要突破这种结构，需要以下策略。

**减少对平台的依赖：**在自家网站发布 PC 版本，向Steam等其他平台拓展。

**减少对广告商的依赖：**通过口碑营销策略、网红合作来增加自然流量。

**不借助IP授权：**开发原创IP，或者寻找休眠IP、小众IP减少IP授权成本。

**降低运营费用：**控制内容开发规模，采用轻量级的更新设计。

这些都会颠覆业界的常识，但是只要不改变这种吞噬利润的隐形成本结构，中小企业就无法生存。

## 第10章：依赖射幸型（侥幸型）模型的局限——与赌博机行业类似

运营型移动游戏的商业模式还存在另一个结构性问题，那就是“依赖射幸型（侥幸型）模型”。这个结构其实与赌博游戏机行业非常相似。

移动游戏的主要收入来源都是“抽卡”，抽卡是一种基于概率让玩家随机获得道具或角色的机制，玩家会为了“中奖”反复行动乃至课金。这种机制在心理学上被称为“变比率强化程序”，这是一种引人赌博成瘾的机制，赌博游戏机也是完全相同的机制。

**依赖部分用户付费的脆弱性**

从运营型游戏的收益结构来看，大部分销售额都来自于极少数用户。通常，付费用户占整体比例不到5%。其中，产生超过 50%销售额的“重氪玩家”，占整体比例不到1 - 2%。也就是极端来说，厂商的收入是由99%的无课金或微课金用户和1%的重课金用户构成的。这种结构类似于赌博游戏机行业，赌博机游戏厅的收入也都是靠常客的“重氪”支撑的。

**监管加强的风险**

这种结构的一个重大风险就是法规监管的加强。赌博机行业在过去几十年中一直受到越来越强的监管，出球限制、兑现限制、广告限制——这些都是为了抑制游戏赌博性的措施，移动游戏行业也正在经历同样的事情。

**抽卡模式限制：**禁止完成式抽卡（2012年）（注：指要靠抽卡获取特定多种道具物品后可将其合成为更高价值稀有度物品的设计）、强制公示抽卡概率（2016年）

**充值上限：**政府正在研究限制未成年的高额充值行为

**海外加强监管：**在比利时和荷兰，抽卡机制被认定为赌博而遭到禁止

**转向“健康付费设计”的必要性**

要避免这种结构性风险，就必须摆脱依赖抽卡的盈利模式。具体来说，可以考虑以下的付费设计。

**订阅制：**按月固定收费，然后可以使用户访问所有内容，没有赌博性付费，被监管的风险较低。

**月卡战令型：**让玩家在一定时间内完成游戏任务获得报酬，保留了一些赌博机制，也保证了持续收益。

**买断制+DLC：**以买断制方式销售游戏本体，将追加内容作为 DLC 进行销售，没有赌博成分。

**加入广告：**可以免费游玩，然后通过观看广告获得报酬，这在超休闲游戏中是主流模式。

但是，这些模式无法像抽卡机制那样获得高收入。抽卡实在是挣得太多了。

在订阅模式下，每月只能收取1000日元左右。但如果是抽卡，有些用户每月会付费10 万日元以上。对于厂商来说，放弃抽卡就等于放弃收入，因此他们无法摆脱依赖抽卡盈利的模式。

但是赌博机行业因为监管的持续加强而不断衰退，机厅和玩家数量都不断减少。移动游戏行业也可能会步其后尘。如果不能摆脱对抽卡模式的依赖，监管持续加强，目前的收益结构就会崩溃，行业不会再有新的参与者进入，最后市场整体萎缩。

## 第11章：中等企业面临的瓶颈结构

不是理想论，接下来要面对现实。中坚与中小企业要想渡过这个冰河期，关键就在于如何面对这些结构性限制。

**资金周转的极限：**只要延期 3 个月就会面临资金链断裂的危险。“试错”的余地几乎没有。

**人才流失的风险：**无法承担长期研发，优秀人才会流失。

**经营判断的僵化：**无法摒弃项目没有推进就等于损失的思想，导致不敢挑战新事物。

**玩家沉没成本固化：**无法从既有游戏夺取新玩家。

因此没办法“等游戏做到好玩”，于是用平庸的 IP 合作或似曾相识的企划来走一步看一步，这就是崩溃的开始。

**依赖IP的陷阱**

“只要使用知名IP就安全”也是一种幻想，中坚企业反而会被IP所束缚：与版权方的内容确认流程会使开发速度降低到1/3，IP粉丝的高期待值会对内容挑三拣四不能妥协，IP授权费固定化无法撤退。结果就会导致臃肿的项目在不健康的结构中运转。

那么中坚企业应该如何应对呢？

**挑战原创IP：**自由度100%，确认流程为零。风险很高，但成功的话回报也很大。

**瞄准休眠IP与小众IP：**谈判空间灵活、期望现实，确认流程也会简便，IP使用费便宜。

**与大公司共创：**专注于开发工作，分散风险，利用大公司的营销力量。

取胜的关键就在于“以轻便、快速、低成本的方式创造独特体验”。

## 第12章：三个结构改革的方面

以下是我对运营型游戏难以生存的结构原因以及其突破口的思考。这不是唯一正确的答案，但实施起来需要一定成本。然而，我从经历过的失败与成功中看到，如果不同时改变组织、开发流程和决策权这三项结构，就无法从根本上解决问题。首先，前提是认识到大多数游戏都需要一定的时间间隔才能钻研出玩法的乐趣，因此在制定企划时需要考虑这一点。

**关键点1：改变组织结构——由谁做，如何做**

**想法1：从三人团队开始。**在大型团队中，“踩刹车”的功能是无效的。但是如果是三人团队（策划、设计、开发），决策会更快，方向转换更容易。在前2到4个月，应专注于用小团队制作“10分钟可玩的原型作品”。如果在这个阶段找不到有趣的核心玩法，无论投入多少人力都是徒劳。

**想法2：将外包方也视为“共创者”。**如果只把他们当作“打工的”，他们并不会说出自己对项目的真实想法。但若将他们视为“共创者”，并为他们的意见支付金钱报酬，他们就会指出设计上的矛盾，通过小额的报酬防止日后更庞大资金的浪费。我所见过的大多数失败项目，都有外包人员察觉了问题却没有表达出来的情况。对外部人员不能仅仅作为承包商对待，营造一个能够自由表达意见的环境，会是决定项目成败的关键。

**想法3：将探索预算固定化。**将年度预算的5%到10%固定为“探索预算”。许多企业都说“没有尝试的余地”，实际上只是缺乏尝试的机制而已。如果有探索预算，就可以以每两周制作一个小型原型的速度进行试验。成果指标不是“完成”，而是“学习”。即使遇到“这种玩法不能由三个人完成”的失败，也会成为宝贵的学习经验。




**关键点2：改变开发流程——要做什么，如何推进**

**想法1：将“10分钟试玩”设为必经环节。**真正有趣的游戏能在10分钟内传达其乐趣。需要30分钟说明的游戏，在市场上很难生存。设定每月一次的玩法评审会议，用“10分钟游玩+10分钟讨论”来判断。如果无法让人产生“再玩一次”的想法，就在这里终止。通过明确这个标准，所有人都能在同一标准线上进行讨论。

**想法2：要分阶段地设置考核点。**

第0-2月：通过原型获得学习经验

第2-4月：通过能玩10分钟的试作品来判断“继续/重做/砍掉”

第4-6月：存活下来的项目可以进行团队扩充

第6月之后：进入正式开发阶段

6个月形成一个周期，只有存活下来的项目才能进入下一轮。10个企划能够通过1到2个即可，其余8个企划要在6个月内停止开发。判断的速度正是中坚企业的生存策略。

**想法3：将改进速度作为KPI。**进入正式开发阶段后，不再以传统的 KPI（如日活、付费率等）作为主要指标，而是以“改进速度”为准。例如，“从确定用户流失点到发布改进版本用了多少天？”这才是真正的竞争力。以小规模测试为前提，可以培养出“不追求完美”而是“快速学习”的文化。没有大公司那样充裕的预算来提升完成度的话，那么只能靠反馈速度来竞争。

**关键点3：改变决策权和判断标准——何时，由谁来踩刹车**

**想法1：将“判断会议”制度化。**现场经常举行进度汇报会议。但没有判断“是否应该继续”的会议。每个月1次，设置一个判断会议来讨论“是否应该继续这个项目？”。要把能说出“虽然已经完成了80%，但并不好玩”的氛围要放在首位。

**想法2：明确“停止权”的归属。**在许多项目中，“谁有停止权？”这个问题没有人能够回答。项目经理、制作人、管理层各自都有难以叫停项目的理由。因此，有必要将停止权明确化，例如设定“在前4个月对项目存续的判断权归项目经理所有”。权限不明确的话，那么谁都无法出手阻止。

**想法3：建立评价“做出停止判断”的机制。**要明确废除“项目停滞导致考评降低”的文化，如果将项目进度与人事考核挂钩，人们就不会说出真实想法。反而需要建立一种能够表扬“初期终止决策”的机制。在4个月内被终止的项目，能够被视作避免了庞大资金损失的功绩——如果不能形成这样的文化，结构性困境就不会改变。

**要令这三个关键点同时发挥作用**

重要的在于同时改变这三项。仅改变组织结构，而开发流程依旧陈旧是没有意义的；仅改变开发流程，而决策权不明确的话，发现问题依然没有人能阻止；仅改变决策权，而组织无法坦率表达意见的结构也是无法运作的。只有同时改变这3个方面，才会诞生“复现出天才的架构”。

## 最终章：将容易发起挑战的环境作为企业文化融入

令运营型游戏度过市场冰河期的关键不是等待天才的降临，而是首先要构建能够复现天才的结构，以此前提组建团队。其中，我认为保证“为玩法的打造留出充足空间→阶段推进项目→正式开发资产”这个流程非常重要。此外，也要注意改善外部合作者的选择以及难以表达意见的环境，不应该仅仅将其视为外包业务。

这篇文章中提到的不只是开发方的问题。玩家的沉没成本、平台税与广告费的上涨、抽卡盈利模式的局限性、市场饱和与差异化的困难，这些都是仅靠开发能力无法解决的结构性问题。正因为如此，才需要改变这种结构。改变组织架构、改变开发流程、改变决策权与评估标准，只有同时推动这三项变革，才能创造出“复现出天才的架构”。

游戏行业要想再次成为“可以挑战的产业”，不仅需要改变产品本身，还需要从根本上改变组织的设计、认知和应对方式。改变结构，这或许是在冰河期中接近商业成功的方法。

## 主机游戏的研发没那么难，和手游端游没本质区别

大家容易高估主机游戏的开发难度，觉得主机游戏技术很难做，而手机游戏和pc游戏就很容易做。

究其原因，无非是见惯了国内团队做手游端游，而很少见到国内团队做主机游戏。而主机游戏的平均水准，远远高于手游端游。

不过这两个原因都不成立。

国内团队不做主机游戏，是因为商业上不成立，做主机游戏能力不够，也赚不到钱。而主机游戏平均水准高，是因为主机第一方厂商的把控比较严。

回答主问题。事实上，有三个因素，导致目前主机游戏的开发，已经不那么困难。

第一个因素，是以前的主机游戏，架构和传统PC不一样，比如PS1、PS2时的工具集和架构，就非常一般，学习成本很高，PS3时代也有SPU这样的架构出现。但现在的主机架构，都比较接近PC的架构，学习成本和多平台的一致研发已经不再困难。

第二个因素，是通用引擎的出现，把不同的主机差异都抹平，只剩下较少的主机开发规范需要遵守。对开发者来说，也更容易上手了。

第三个因素，是随着性能过剩，大量非顶尖游戏，并不需要充分发挥主机所有的机能。这样一来，很多深入、针对主机架构的优化也就没必要做了。而优化在传统主机游戏开发中往往是非常重要的。

当然主机开发还是有自己的难度，不同于传统的端游手游开发。

第一个难度是，主机开发者的准入门槛。要开发主机，你必须有厂商给予的开发授权，以及配套的开发机。而这个授权并不好拿。近年来三大厂商都加强了对独立开发者的拉拢和扶持，门槛才有一定降低。

第二个难度是，主机的开发规范很复杂。都是很多琐碎的细节，但当主机厂商规定了几百页的规范细节，就够开发者喝一壶的。比如游戏中任何时间不能有画面停滞这一条，就要做到死。所有的Loading画面，大量高时长的卡顿，都需要额外开发一些动态的元素，表明没有死机。新开发团队容易忽略这些开发规范占用的开发时长。而主机厂商对于游戏crash、对于性能，都提出更高的要求，普通手机或者端游应该远远达不到这个标准。

第三个难题是，主机厂商的品质管控严格。主机游戏之所以平均质量高，离不开主机厂商对游戏品质的把控。对于综合素质不高的产品，主机厂商往往不会通过审批。这个审批和国内政府审批又有所不同，国内政府管的是政治正确，而主机厂商管的是产品品质。近年来门槛似有降低，但比起随便就可以上线的手机、端游，还是严格不少。

不过，在一些方面，主机开发甚至是更简单了。比如兼容性问题，就是一个很好的例子。主机机型固定单一，不用做那么多兼容性。一个不恰当的例子，相比于安卓平台的碎片化，做iPhone版本的手游，可能就会愉快得多，配置简单，问题就会少很多。主机开发相比iPhone游戏开发，就更简单了，基本不用考虑兼容性问题。

综上，主机开发难度不应被高估，它就相当于一个普通的开发平台。

## 国内的研发成本并不低

大家总觉得国内研发成本低，国外研发成本高。

由于过去这些年国内游戏行业的迅速发展，国内一款游戏的研发成本已经不低了。

前些年手游浪潮一起，市场上充斥着大量只工作过一两年的游戏开发者，由于缺乏开发者，他们的薪酬还特别夸张。

这两年随着市场成熟，逐渐开始回归理性。但即使如此，如果考虑人的能力因素，国内的成本还是不低。同样的收入，在海外能找到多项全能的开发者，而国内往往只能找到能力单一的开发者。

如果用国内大厂的薪酬水平来衡量，那么单一开发者的平均薪酬，相比发达国家，也不算低。东南亚、东欧、俄罗斯等地区薪酬很低，欧洲、日本开发者的平均薪酬是比较低的。再考虑类似有些地区政府还在扶植游戏产业。加拿大政府对游戏产业补贴，在这个不正当竞争条件下，在加拿大开一个工作室的成本空前的低。放眼全球，也只有美国开发者平均薪酬比国内略高了。

## 引擎等中间件技术不赚钱

引擎技术作为游戏开发技术中的核心要素，有较高的技术含量。10年前的高端商业游戏引擎，价格非常昂贵，都是几十到上百万授权费用，外加收入分成条款。

即使有这么高的授权费用，引擎公司生存条件依然很艰难。开发cryengine的crytek，几度徘徊在破产边缘；开发unreal的epicgames，长期以来业绩也是依赖自己的游戏产品在撑门面；gamebyro这样的公司，技术层面并不如前两个引擎强，逐渐就淡出这个行业了。

unity改变了游戏规则，把卖授权的价格降低到白菜价，致力于打造开发者生态，卖的是服务。我没有unity这里的业务数据，但这条路，显然也很艰难，比不上做个爆款游戏来得划算。因为一个很简单的道理，大量帮助优化、提供高端支持、培训服务等技术增值服务，都依赖于技术人员的人力投入，提供这些服务的边际成本，并不能随着规模增加而快速降低。

作为高端技术提供者，引擎厂商已经很难过日子，而中小型中间件提供商就更艰难了。大多数组件，都很难坚持卖很久，很多中间件最终都结束了独立的状态，被其他大厂收购，成为大厂软件生态的一部分。能活下来的小厂商，一般都有两个特点，一是相关领域比较狭窄，利益不大，没有竞争者；二是这些厂商心态也比较好，不着急扩大规模。一旦跳出这两个圈子，马上就不行了。

## 不是使用了UE4引擎，就能做出AAA游戏

很多团队开始要准备做AAA游戏。团队信心十足，先切换UE4，一看Demo美术效果惊人，就觉得一定能做出AAA游戏。

从Demo到游戏的差距，三年前写过一个系列，可以参考：

[顾煜：为什么越受重视的游戏项目越难开发好（1）](https://zhuanlan.zhihu.com/p/20200956)

UE4有UE4的好处，但做大型游戏，并不是用工具就能解决所有问题，人的因素也很重要，倚天剑真到了厨师手里，也未必会比菜刀更好用。

开发者团队才是最重要的因素，不要看他们用了什么引擎，要看这个团队做过什么质量的游戏。

当有了足够好的人才储备，切换什么引擎，无非就是多些学习成本，多点时间投入。

## 做引擎就是做渲染

在那个遥远的过去，那时候Unity还没有统一江湖，Unreal还没有被高端开发者接受，彼时大家真诚地认为，做渲染就是引擎的全部。

那时候的自研引擎，都是只有一个渲染模块。那时候的商业引擎、开源引擎，也有很多只提供了渲染模块。

这是一个很大的误解。引擎是一个基础，就像汽车底盘是所有功能的承载。在这个基础上，需要开发大量其他特性，渲染只是其中一小块。我们还需要做跨平台特性、对象生命周期管理、资源管理能力、工具链管理、内存管理、多线程调度、动画管理模块、物理模块等等等等。

好在那个时代过去了。现在大家都习惯了商业引擎，也没多少人再去做引擎了。

于是连带着，也没多少人做渲染了。。。

会，甚至可以说已经饱和了。

但从本质上说，这是独立游戏作为一种身份标签的饱和，而不是创意的枯竭。

![](https://picx.zhimg.com/v2-cc38500b5fea0f0ded0987a256a35241_r.jpg?source=2c26e567)

根据HushCrasher的统计，「独立游戏」在Steam Greenlight启动后经历了数量级的暴涨，在Steam Direct后发行量继续上升了一倍有余，但获得的关注度按评论者比例来看却变化不大。

供给爆炸和注意力停滞之间的矛盾，显然反映了一种「饱和」。

![](https://picx.zhimg.com/v2-c5a9d8d9ff1a6d4f422f15ee4b336abe_r.jpg?source=2c26e567)

事实上，独立游戏已经大幅偏移了其原始含义，主要服务于一种媒体塑造的「反游戏工业化」、「反3A」的叙事符号。当这种符号被过度消费，玩家自然会开始觉得「独立游戏越来越多，又越来越同质化」，实际上「或许并不能提供新创意和新体验」。

这可能**不是独立游戏本身缺乏创意，而是独立身份失去了稀缺性和区分度。**玩家逐渐不再把「独立游戏」视为一个有效的信号。[1]

从产业结构也能看出独立游戏本身就具有很大的模糊性，比如Steam上的大型3A游戏里有1/4由开发商自主发行，而独立游戏标签下的游戏超过1/4有发行商。

![](https://pic1.zhimg.com/v2-e946f6776f575771608fe4fa798638da_r.jpg?source=2c26e567)

这样一来，「独立游戏」很难再称得上是反映了实际的生产关系，而是愈发成为**在与「主流」的对比中被定义的、被媒体反复建构的一种身份**——意识形态意义甚至比定义都清晰。

日本游戏的环境进一步证明了这一点。表面上看，日本似乎缺少「独立游戏」，但这**只不过是日本难以提供这种反叛叙事的想象空间**：小型商业公司与同人社团的运作方式差距不大，「独立气质」的作品在日本也不以「独立」作为卖点，商业和「独立」形成的是粉丝互文而非反叛对立。
——而不是日本真的没有「独立游戏」。

[为何近几年出圈的独立游戏多半来自欧美、中国，但很少听到日本有很火的独立游戏？](https://www.zhihu.com/question/1961723161348379743/answer/1964385818731848388)

但这其实和行业前景关系不大。「独立游戏」仍会发展，小团队用有限资源尝试新玩法、个人化表达的作品仍会有吸引力，只是独立游戏标签的叙事失效而已。

最后，「独立游戏」成为「反游戏工业化」、「反3A」的叙事符号，其实本身也是一种「3A本位主义」。
「独立游戏」和「3A」的对立其实建立在PS3世代后中型游戏（AA）的大规模消失上。[2]在PS2世代，创意与商业并存的中型游戏才是当前独立游戏的生态位，甚至更好：预算相对有限但仍然完整，小团队制作但有商业化逻辑，有创意但不脱离市场。**这种「适度个性表达」的消亡才是游戏多样性出现塌陷的根源**，结果媒体和玩家将创意表达的希望全部投射到「独立游戏」上，进而形成了过度期望。
但是当3A无限膨胀已难以为继，叠加上现代工具链的易用性，也许我们真的能在不远的未来看到中型游戏重新兴起。那么，不再需要承担「反3A象征性角色」的时候，「独立游戏」标签也就不再重要了。

最后，Sketchup 还有光照系统啊！不管贴图多简陋加上光照阴影感觉立刻就高大上了啊，而且根本不用花时间摆放光源什么的，设置一个季节和时刻直接就可以看到效果了啊！

第一，我们是一个随时替换的工具

第五，你可能没有一个好的规划，也不知道未来的发展方向，也可能没有一个设计领域的大神作为榜样。**所以你后面的路可能走的会比较迷茫，**（设计是一个奇怪的行业，大咖撑死，有点名气能力的可以吃饱，但是下面没有什么名气的可能要饿死，不像别的行业年龄越大会的技能越多越不可取代，设计师年龄大了直接干掉找个便宜的年轻的，毕竟丑点不要紧不要觉得企业会在这些东西上花多少钱，没有设计又不是做不下去不能用，无非就是丑点而已）

第六，这行能坚持下来的真不多，发展潜力的天花板太低了，能混的好的真不多，坚持久了的人又舍不得放手一身本事，又不甘心这微薄的待遇，慢慢3 5年 年级大了反而越来越焦虑

这是我的自问自答，《怎样成为一个给力的游戏UI设计师》

副标题，神仙道从网页到手机经验谈之UI篇

如果要做好一件事情，我们必须对它寻根究底。

首先我们需要定义下“游戏UI设计师”。

因为“游戏UI设计师”不是“游戏设计师”，也不仅仅只是“UI设计师”。

我们必须搞清楚它到底是什么，这样才好进行下一步。

习惯性的在wikipedia里寻找答案，

不过…，很抱歉这次wikipedia中文里没有现成的答案让我偷懒

（PS：别问我wikipedia英文里有没有，我E文很烂全靠老婆，不过现在凌晨3点50分，她在睡觉。）

好吧，我们用笨一点的方法进行。把“游戏UI设计师”，拆成“游戏”，“UI”，“设计师”3个词来寻找答案。

**游戏**，人的一种娱乐活动，也可以指这种活动过程。

**UI**，（User Interface ，也称使用者界面）是系统和用户之间进行交互和信息交换的媒介，它实现信息的内部形式与人类可以接受形式之间的转换。

**设计师**，是对设计事物之人的一种范称。

通过上面3个词的定义，我们可以总结得出：

**游戏UI设计师**，是指通过设计界面让游戏系统和玩家之间能进行互动娱乐的人

好了，理论结束。我是草莽之徒，相比理论更讲究实战。

接下来让我们以【神仙道手机版】的开发过程为例，说说游戏UI设计师。




首先，项目背景：

2011.7 神仙道网页版获得成功

2011.8.23，大家决定移植神仙道网页版到IOS上

2011.12.22，神仙道IOS版 Version1.1发布，坦白说体验并不好

2012.2.18，在神仙道IOS版 Version1.3.4时，大家接受了新的UI设计思路进行调整

2012.5.3 ，发布了神仙道IOS版 Version2 ，效果显著。

2012.5.12，发布了神仙道IOS版 Version2.1，补充了很多游戏内容。

之后神仙道IOS版稳定更新，很快在app store畅销榜爬到第一，并呆了很久。

（PS：光荣属于团队的每个人，这篇文章仅以游戏UI设计师的视角叙述。）




从项目背景可以得知，手机版的需求是在网页版成功后产生的。

这意味着什么呢？

哈，游戏设计之初完全没有考虑过手机版！




好吧，我喜欢的作家九把刀说过“人生就是不停地战斗！”

并且他还说过“因为无所畏惧，即使时不我予，英雄也能逆天而行。”

既然喜欢的作家都这么说了，那没办法，硬上了。




将网页游戏神仙道移植到手机上的**第一个问题是“神仙道网页版内容真的太多，而手机的画面太小。”**

只有对网页版的游戏内容进行筛选，保留主干去掉不必要的系统才有可能实现移植。

这要求游戏UI设计师要非常熟悉游戏的结构，特别是对游戏内容的熟悉。

# 日本非知名漫画家的收入与日常生活是什么样的？ 刘石舟

**Author:** 刘石舟  
**Bio:** 品牌市场/农业金属合作社/赛虎抄手/

蟹妖

现在日本的漫画出版的大环境并不好，从21世纪的第一个十年开始，日本的整个漫画杂志的发行是萎缩的，年发行量大概不到90年代全盛时期的2/3，原因有二：1.日本“少子化”现象越来越严重，作为漫画主力消费人群的小孩儿越来越少；2.电子游戏、智能手机的冲击。连载漫画的发行越来越少，三大社都把更多的精力放在小风险利润高的单行本上。当然接踵而至的问题是，日本的愿意画漫画的年轻人也越来越少了。

在日本，漫画家很大一部分收入来自于版税，一般每卖出一本单行本从中抽出5%到15%的版税，那些销量过亿的作者，版税收入是非常多的，2013年，尾田荣一郎（31亿日元）、岸本齐史（18亿日元）...富坚义博在93年纳税就已经达到3亿日元，青山刚昌96年以来一直是高额纳税人排行榜的常客...这些家伙长期占据在日本漫画收入金字塔的顶端，不可动摇。**往下走**，第二档的漫画家的年收入大概是10亿日元，接近6000多万人民币。以谏山创为例，2013年10月，《进击的巨人》单行本发行已经突破2300万册，单行本售价440日元，如果按每本版税40日元计算的话，谏山创去年的收入妥妥突破10亿日元无障碍。**再往下走**，一般的漫画家，可以做到连载的话，一般是1万日元/页的稿费（一线大神们单页价格要高很多），年收入可达[500-1000](tel:500-1000)万日元左右，大约是30-60万人民币。如果连载有人气的话，就可以做单行本，就能享受版税了。单行本漫画一般出版至少3万部以上，有人气的话就可以出版10-20万部以上，妥妥实现屌丝的逆袭。



![](https://picx.zhimg.com/50/26f543bec14452578c3d9f4b1c59b58c_720w.jpg?source=2c26e567)

老一辈少女漫画家齐藤千穗的豪宅

然而构成金字塔底端的绝大多数年轻漫画家，如果没有连载或者连载被砍的话，必须要一边打工糊口一边继续画画，或者做著名漫画家的助手。之前网传的《少年Jump》招募连载助手的启事：包住宿，日薪1万到1万7千日币（09年），支付交通费，伙食费，正职人员享有单行本红利。

虽然日本漫画从业人员趋于减少，但是伴随着杂志出版的萎缩，职业助手市场竞争仍然非常激烈，由于雇主漫画家的收入水平不一，助手个人素质不一，干的活也有复杂和简单，因此时薪跨度可以从7万-30万日元，但由于并不是一直都有好活干，因此大部分助手的在漫画方面的进项都在150-300万日元（日本普通职员的平均年薪在370万日元之间，而日本大学刚毕业的学生一般都能拿200万以上年薪），基本可以说单靠做助手在东京是无法养活自己的，只能去做别的兼职，还有的可以同时为两个漫画家做助手。同时由于漫画家的工作强度非常大，除去一两个固定的主力助手外，其他打酱油的助手们辞职频率非常高。不过，如果有实力和运气能成为名漫画家的主力助手，月收入50~60万日元都没有什么问题。



![](https://picx.zhimg.com/67f4d09a5ad4cb8ca6cc00fa2497e150_r.jpg?source=2c26e567)

谏山创…自宅

当然并不是每一个漫画家都请得起助手，截稿日之前，无论有名与否，都必须拼命赶上编辑的要求，连载期间每周工作6天每天工作12个小时是基本的要求了。井上雄彦在《Slamdunk》连载之初，还需要一边为北条司打工来赚钱糊口，一边画自己的画，哪怕到了SD开始走红之后，他还是在相当一段时间内请不起助手，所有事情亲力亲为，每天都需要画十几个小时，直至经济状况得到改善，才一次性请了4名助手（这也是为什么SD到了第七话中段画面突然精致起来的原因）。

![](https://picx.zhimg.com/a5b44bf34ef8c3051c62fcd2e6e651bd_r.jpg?source=2c26e567)

桂正和的工作台


日本的漫画家基本都集中在东京，但是因为时势的变化，早已没有常盘庄那样漫画家聚居的地方了。一线的漫画家大多购买或租赁了高级的公寓或别墅，助手们和编辑们可以直接上门来工作，牛逼一些的还会在出版社左近租工作室，更牛逼一些的还会把工作室改为他用（比如浦泽直树的工作室后面便是他私人的琴房和收藏室）。

**2014.7.17第四次更新****提前说明：本人并非专业研究漫画，只是对这方面比较感兴趣，所答内容仅供参考，如有错误，还请指正。**

**以下主要针对日漫。**

**先放出结论：不能一概而论。**

原因是，不同漫画与漫画家的创作方式有所不同。

以最典型的日本漫画而言，从大类来分，大致可以分为以下三种。 (由于日本漫画基本均为杂志连载，因此不讨论其他类型漫画）

**第一种模式，也是最普遍的模式，即漫画家自己负责创作剧情与作画。**大部分日本漫画与网漫均采取这种模式。这种模式需要漫画家本身具有一定的叙事能力与较强的绘画功底。我们熟悉的如《海贼王》（尾田荣一郎）、《火影忍者》（岸本齐史）、《死神》（久保带人）等作品均采取这种模式进行创作。

**第二种模式，即专人（原作者）负责剧情创作，漫画家负责作画。**这种模式能够发挥两个人的专长，使其各司其职，使漫画的创作达到最佳表现效果。负责剧情创作者同样有两种模式：通过文字表现剧情与通过画面表现剧情。前一种模式一般是由原作者写出脚本，再由作画者绘画。对于后一种模式而言，通常是原作者先画出分镜稿（仅仅包括了主要剧情，不要求作画效果），再由作画者进行重新作画，画出原稿。这方面最有名的例子无疑是《死亡笔记》（大场鸫原作，小畑健作画）与《出包王女》（长谷见沙贵原作，矢吹健太朗作画）。（关于这方面的更多信息，可以参照

[@刘石舟](https://www.zhihu.com/people/23e3d30e0bb2135ca06dfe876d183e9c)

的答案，在此一并表示感谢）

**第三种模式，即所谓的改编。**改编的来源可以是动画，小说或游戏等。由于原作剧情已经明确，漫画家自由发挥的空间较少。当然也有这种模式的例子有改编自小说的《杀戮轮回》，改编自游戏的《寒蝉鸣泣之时》，改编自动画的《境界线上的地平线》等。这也是日本ACG产业流水线化的体现。也有根据其他漫画进行二次创作的（即所谓的重制），即将原作剧情重新改编作画，常见于一些网漫，典型的有《一拳超人》（ONE原作，村田雄介作画）与《堀与宫村》（HERO原作，萩原大辅作画）。当然，也有《PLUTO》（手冢治虫、浦泽直树）这样仅仅借用原作世界观与人设，剧情全部重新创作的作品，此为后话。

**但是，无论如何，始终有一点需要明确：杂志社的责任编辑在创作过程中会发挥重要作用。**一切创作都要经过责任编辑把关，发挥其审稿人的角色。实际上，责任编辑在某种程度上对剧情与作画有决定性作用，特别是在类似于集英社这种竞争较为激烈的杂志社。举个例子，《金田一少年事件簿》早期由讲谈社编辑天树征丸提出创意，由金成阳三郎编写剧本，由佐藤文也作画。但在后期基本剧情全部由天树征丸提供。这无疑得益于他编辑的身份，随后他也转型为原作者。

**不过，目前日本漫画出现了一个新特点：漫画家带领团队进行创作。**最典型的例子是《名侦探柯南》（青山刚昌）。此人把团队协作发挥到了极致，个人只提供主要创意与剧情走向，而由助手进行剧情加工与作画。这种方式往往为圈内人所不齿，但是不可否认，这样做集思广益，很有效果。事实上，多数国漫现在采取的正是这种创作形式。个人以为，这种创作方式无可厚非，仅仅是一种新的潮流而已。

其他漫画类型个人了解不多，还请高人指点。

![](https://pic1.zhimg.com/v2-fb02e54ba5d79f7d4fa83f122d067d99_r.jpg?source=2c26e567)

![](https://picx.zhimg.com/v2-5c1df4815700b6aa40e2d37312c55ea5_r.jpg?source=2c26e567)

livid的观点很明确：v2ex是他的个人站点，他向用户提供服务，同时也具有最高的执法权，并且没有义务对他的执法方式作出解释。他认为不合适的（或潜在不合适的）内容或用户，他都可以直接删除，无需解释（解释是无意义的，会浪费他的生命），因为v2ex是他自己打造的空间，**一切解释权最终归他所有**。

该贴下面的回复中，绝大多数支持livid的理念，认为v2ex的用户就像在livid家里做客一样，客人让主人不爽，主人随时可以把客人从家里赶出去；小部分回帖表示这样的做法欠妥。

livid的想法是可以理解的，因为他和v2ex用户并不处在平等的关系中：他负担站点的开销和维护，让站点能正常访问，站点的用户仅仅是在消耗网站资源，对livid而言并不能增添什么积极价值。如果一切安好，那就ok，v2ex就正常运作下去，如果有刺头不老实，吵着要什么「解释」，那抱歉，你就是潜在的消极价值携带者，直接ban掉不解释。

所以，v2ex的用户们应该有这种觉悟：**你们的存在并没有给v2ex增光添彩，而无非是给livid的「个人站点」添负担**。他之所以没有赶你走，是因为你现在带来的负担仅仅是隐性的；当这种负担显露征兆，你就被判定为有害的，需要被清除。

豆包手机的AI助手能干什么？它可以跨应用操作，看你屏幕上的所有内容，帮你点击、输入、跳转。听起来很酷对吧？但问题就出在这，它打破了移动互联网最基本的游戏规则。

首先说句公道话，它们封杀豆包，在现有的技术框架下不仅合理，甚至可以说是必须的。

目前的所谓「AI 手机」或者「自动操作」，并不是真正意义上的系统级底层互通。因为 Android 系统的沙盒机制（Sandbox），应用之间的数据是隔离的。豆包要想操作淘宝或者银行 App，它主要依赖的是安卓的 AccessibilityService（无障碍服务）。

这东西本来是给视障人士用的，能读取屏幕内容、模拟点击。但在风控系统眼里，这就是个巨大的「后门」。

具体的安全隐患我就不细说了，这个搞安全的比我专业的多，我就举一个例子，完全可以用同样的接口写个脚本，绕过人脸识别把你的钱转走。所以银行、微信、支付宝这种涉及到钱的问题的app封它是技术本分。

置于王者荣耀我觉得就是胡扯，不过如果非要往这上面靠那可以说是避免充值的问题，也不是说不对。

其实这里涉及到一个流量入口与「去中介化」的争夺：

现在的互联网商业模式，建立在“注意力经济”之上。你需要打开淘宝，看开屏广告，刷首页的信息流，被直播间的主播忽悠，最后冲动下单。这个过程叫转化漏斗，每一个环节都是钱。

试想一下，你跟豆包说：帮我买箱最便宜的维他奶。AI 直接后台比价，绕过淘宝首页，绕过所有广告，直接调用接口下单。 在这个流程里，淘宝从一个万能的商业帝国，瞬间退化成了一个单纯的数据库 和 物流接口。 这种现象在 Ben Thompson 的聚合理论（Aggregation Theory）里其实早有预言，谁掌握了离用户最近的那个界面，谁就掌握了话语权 。以前这个界面是 App，现在豆包想取而代之，那你说你都这样了，不干你干谁呢？对吧

其实这事腾讯是最熟的，它们在23年就发了论文了：

[https://arxiv.org/abs/2312.13771](https://link.zhihu.com/?target=https%3A//arxiv.org/abs/2312.13771)

以前回答我都懒得发图，今天特例发个截图，你看论文第一张图，一目了然，对吧。

![](https://pica.zhimg.com/v2-ea00bf3df0bce36e53860a55aee0cb9b_r.jpg?source=2c26e567)

论文里明确指出了一个点：目前的 AI Agent 是不需要 App 提供专门接口的，它就是通过视觉感知（看屏幕）和模拟操作（点屏幕）来硬解。这里面能干什么它们能不知道吗，门清。所以直接第一个出手，直接还是2选1，反正就是让你用不了。 以前防爬虫，防的是 API 接口被刷；现在防 AI，防的是GUI（图形用户界面）被理解。 一旦 AI 的视觉识别能力突破临界点，它就能无视 App 的 UI 变化，像一个不知疲倦的「全能管家」一样接管手机。这才是大厂们最怕得，因为他们掌握得流量入口就没了。

所以以资金安全为理由直接下线，这个理由非常冠冕堂皇，但是你又不得不说这个理由没毛病。

这次被围殴，其实是把 AI 行业那个秘而不宣的尴尬摆到了台面上： 在没有搞定利益分配机制之前，AI 越智能，由于它对旧生态的破坏力越强，它受到的抵制就会越猛烈。

这场冲突短期内很难有好结果。豆包手机现在核心应用都被封了,实用价值大打折扣,可能就沦为极客圈子里的玩具。中期来看,要么字节跟各平台谈判妥协,开放官方API接口,AI只能在"许可范围"内操作;要么就是技术军备竞赛,豆包想办法更隐蔽地模拟人类操作,平台用AI来检测AI。长远看,这事儿甚至会倒逼行业标准和监管介入，或者出台专门针对AI Agent的数据采集和用户授权规范,让大家有章可循，但是你信么？

现在是下午一点多，可能是早上起来没吃早餐也没吃午饭光喝咖啡了，突然我浑身发抖虚弱无力，可能是低血糖了。于是我叮咚买菜上买个个西瓜跟几盒牛肉之类的凉菜，半小时能送到，我一边等一边写吧。要是写一半断了可能就是饿死了，你们帮我找叮咚买菜索赔。

喝口七喜继续。

有一天，网上出现一条消息，有个叫梦想家的网站要搞活动，在北上广各召集4个人，把12只小白鼠关到宾馆里72小时，不许打电话不许跟人接触，只靠一台能上网的电脑，看能不能生存下来。这么好玩的活动我得参加啊，上传了几张照片，发了一段自我介绍我就报上名了。选人的规则是由网民来评选最**喜爱**的人。在几千个报名者里面选12个，广州有4个名额。作为当时广州网民最帅的一个，或者说最帅的之中最能搞事的一个，我发动了所有能发动的网友帮我投票，很快我的得票数就在广州区遥遥领先，即将获得其中一个名额了。

如无意外的话，这时候该发生点什么意外了。这时候出现了一个智障——对的就是智障，就是有点脑子但不太多。

这种投票，选的是受群众欢迎的网民代表，就是网民觉得你行你就行，我周围亲戚朋友同学同事能上网给我投票的人多，我再拿个大喇叭上街拉票，能拉到人给我网上投票，就说明我有本事对吧？那我自然是受网民欢迎的人没错吧。

我当时是网易BBS的一个版主，对，就是北有水木，南有网易的那个网易，你别管信不信，反正当时上网的你不知道网易BBS你就不时髦。照现在的标准，相当于在小红书和抖音B站排前三十的网红，里面有我。我振臂一呼能鼓动几千票把我选为黎明第二。

就这么个情况。在投票截止前三天，出事了。有人得票数超过了我。

也不是不行，你就说黎明，木村拓哉，谢霆锋，吴彦祖，黄渤，还有我站在一起，颜值我估计排不进前三，但你要把我挤出前五，那肯定有猫腻了对吧？

票数超过我那人，从资料介绍上看是铁路系统的一个信息技术人员。他的得票数字很奇怪，五十五十的往上涨。是一次加五十，过几秒再加五十。等分钟吧，再加五十，很整齐。

那个时候互联网络不太稳定，经常会有延迟断网之类的情况发生，所以这种情况有时候也没办法。过了二十多年，我还能想象到当时他坐在电脑前看着票数，时而会心一笑的表情。

所以在投票截止前几分钟，这个人得票数掉到了第五。网络嘛，那也是没办法的事。

他又没有黄渤那种才华。

于是我坐上飞机到了上海。

上海华亭宾馆，来自广州和北京的各两名选手进入了生存测试的房间，每人除了穿在身上的一身衣服，只有一卷卫生纸，面对的是一张光板床和书桌上的一台电脑。这四个人将在千万网民的注视下表演一场为期三天的真人秀。

吃累了我躺一下子。

大家好我回来了，我没睡觉，吃饱饭我挣钱去了。这两天我吃的喝的都是网上买的，没进过线下商店，你们呢？

继续。

游戏规则是：在一个酒店标间内，放置一张光板床，没有床单枕头等床上用品。一套桌椅，一台能够上网的电脑。房间有电，卫生间有自来水，有一卷卫生纸。

参加测试的人员得到一个银行卡的卡号和密码，卡内有1500元人民币余额。另有1500元额度的现金掌握在门外的工作人员手里。房间里装了摄像头（卫生间除外），测试者的一举一动实时在活动网站直播。

我这人比较实在，一门心思照着“生存”的模式去玩。我给自己设了个场景，在茫茫的大海上，我困在了一个孤零零的小岛，我需要有能喝的净水，有食物能填饱肚子，能御寒的衣服，弄到基本的工具和武器，还要能向外界发出信号宣示我的存在并且求救，这一切只能通过外星人（划掉）失事轮船上抢救出来的一台电脑来完成……

醒了。你们想看的没有吃喝那种场景不存在，就算有我也不会写出来。就像当时的直播镜头里不会出现测试者洗澡上厕所的镜头，儿童不宜的场景更不会有。

在1999年，中国人要买东西只能走出门到大街上，根据电视里，报纸上的广告来指导消费。偶尔从杂志上看到有邮购产品，会到邮局往外地的某公司汇款，汇款单附言里写上你要买什么东西。几个星期后那样东西就通过邮政包裹寄到你家里了。

所以我从确定参赛的时候起，就知道荒岛生存模式是玩不通的，那样只有一死。活动组织者也清楚这一点。

但有一点是确定的：如果我们已经能够把广州到北京，上海到纽约的通讯从几天缩短成几秒了，那有没有可能把以前一个月才能从香港送到北京的进口电视机，变成当天送到？这个测试能不能做到？能不能至少让更多人通过这个活动了解网络，知道网络的新奇好玩，让所有人都来参与和思考，网络都可以做什么？

这就像人类发明了火车和飞机，肉身移动的速度增加了几百倍，发明了电脑，运算速度增加了亿万倍，互联网一定也可以改变些什么，让我们的世界有亿万倍的变化，让人类的能力有亿万倍的提升，直到星辰大海~~~~

我带着对互联网的美好畅想走进了房间。

我知道你们想看什么。

是的，一共12个人，有一个在第二天就出来了。他没法在网上买到任何东西，心理极度挫折之下开门认输。这个结局其实主办者并不意外，他本来就是设计好的节目效果之一。很多人未必记得这个生存测试的细节，但都津津乐道有一个失败者不会上网买东西，提前被淘汰了。

现在来看，一个没有电脑使用经验，不会使用智能手机的老人，在现在的社会，可能比他好不了太多。所以我们还要有市场，还要有能够使用纸币的收银台，还要有不依赖网络能够生存的社会环境。建国后我们花了几十年普及教育，消灭文盲。因为文盲在现代社会几乎寸步难行。三十年，五十年后，不会使用电子通讯工具的人也将寸步难行。

这个活动当时的报道基本上没有夸张的地方，人们记忆中的基本上是事实。在活动举办的时候，没有饿了么和美团，没有京东，没有淘宝。现在我们在使用的所有购物平台，当时都不存在。当时有什么呢？

[http://www.8848.net](https://link.zhihu.com/?target=http%3A//www.8848.net)，一个以卖软件为主的购物网站。老板王峻涛先生，是中国电子商务的先驱之一。他另一个广为人知的身份是“大连金州不相信眼泪”的作者。当时8848的购物模式是，你给他们的银行账号汇款，附上你的地址。他们确认到账后把软件光盘或者磁盘通过邮局寄给你，要是你愿意多花几十块钱运费，他们也可以加快，用EMS包裹寄给你，一般国内大中城市三天送到。

……

没有了。

对，没有了。我进测试房间之前，确定能买到东西的，只有这个网站。他是卖软件的。是卖软件光盘和软件磁盘的。还有一些电脑工具书，不卖吃的。

除此之外，我不知道哪家网站能买到东西，并且是能在三天内送到。

所以活动前主办方轰轰烈烈的宣传，把基本上所有有影响力的媒体都拉了进来，除了网络确实是一个新鲜事物外，大家也怀着忐忑的心情在做一件事：大伙听好了，我们要把12个人发到荒岛上，大家多想想办法给他们扔点吃的喝的，什么都行啊别让他们死了。

有很多人听到了，但还在观望。这些人包括永和豆浆，包括北上广三地的大型商场，比如广州百货大楼，北京城乡购物中心之类的，包括网易新浪搜狐几个门户网站的购物频道。包括当时不知名现在也不知名的合法或不合法的网站和一些传统商铺。

所以现在来看，测试过程中购物环境是很混乱的。

要买东西，首先要知道：在哪里有卖呢？

现在我们要买东西，直接打开几个购物app就可以了。但当时没有。于是，测试开始后，活动的专属网站上，我们就开始发帖：

我是测试者，我想问问大家，北京哪里能买到可口可乐呀？有康师傅泡面的话来一箱更好。

我想吃烧鸡，我现在在XX宾馆，大家懂的。没有烧鸡包子也行，门口工作人员会帮我给钱，记得带发票。快点哦饿了。

啦啦啦啦啦啦我要二十只大闸蟹送到二环路19号三楼，快点哟。

有没有人能帮我买一个麦当劳的巨无霸啊？好想吃。最好是加大套餐。我在XXXXXX。

哇哈哈水来一箱，货到付款XXX宾馆XXX房间。记得带零钱我没零的。

空调好冷我需要一床被子。谁家有卖床上用品的帮我送一下，还要一个枕头一个床单。货到付款。（有洗脸毛巾和香皂一起带过来）

在这些专为活动特制的网站里，多数商品是为活动特设的。此外也开设了留言板供“客户”留言。

比方说，永和大王的购物频道会有这样的留言：我想吃你们的红油馄饨能不能今晚和明早各送一碗，再加一份煎饺多放醋。另外帮我买一包一次性内裤好吗我要大号的，钱一起算。我是测试者XXX在808号房。

8848餐饮测试频道里会出现这些：我买一份windows2000中文专业版，帮我顺便带一箱可乐。XX宾馆906房间请快点我要渴死了。

当然，也有热心网友自发给测试者打气的，我的房间门口就有朋友送来的气球，下面有一封手写的鼓励信。

有些办公设备公司，会主动联系我们：建国饭店参加测试的好汉，我想赞助你一台笔记本电脑，能不能在镜头前喊一下我公司的名字？

有些围观的网友出于好奇定了东西，会接到网站的电话：“不好意思我们只给参加网络生存测试的选手送货。”

活动就这样进行着。目前为止，算是解决了第一个难题：怎么找到我想买的东西。即使这个问题是以信息定向发送的方式解决的。

有人可能会说：当时明明还给了你们一张银行卡~~~~

对了，每人有1500块钱现金，根据测试规则，现金由门外的工作人员保管。商品送到后，工作人员代为签收，付款，把东西放在房间门口敲一下门，躲开。测试者开门把东西拿进房。整个过程中测试者不得与任何人发生肢体/语言/眼神接触。

那每人还有一份银行卡里的1500块钱呢？这就好玩了。

当年，确实有些渠道是可以接受电子付款的。比如8848。付款流程是这样的：我先下载一个某银行的电子控件，比如说建设银行，需要先到柜台签几张纸质协议，银行给我一个密码纸条。我就回到我的电脑上，下载一个建设银行软件包。再打开8848网站，点击购买软件，到付款页面，跳转到建设银行界面，输入我的身份证号码，输入银行给的电子支付密码，输入8848公司的银行账号，点击电子汇款，银行反馈给我一个确认：已从我的账户将XX元人民币转出，一段时间后汇入8848账户。

然后我就等着。

等到某个时间点，可能是当天下午，可能是周五，8848的财务有空了，查一下他们的帐户，发现多了一笔汇款，就把这笔汇款跟业务部门的销售记录比对，确认无误后，8848的仓库给我发货。

跟现在淘宝买东西相比，好像也不是很麻烦对吧。

但是你怎么保证8848会给你发货？

就算你相信8848，你怎么保证另一家卖枕头的会给你发货？

你怎么相信卖电脑的在收到你一万块钱后会给你发货？

你怎么相信一个网站上写着：北京购物中心，网址是[www.happy666.com](https://link.zhihu.com/?target=http%3A//www.happy666.com/)的网站，在收到你五千块的冰箱款后，会给你发货？

这其实跟我看了故事会上的增高药广告，给某保健品经营部汇款八百块，最终没收到货一样。把购物行为从邮购换成网购，把钱从邮局汇款变成网络转账，其实并没有改变什么。因为在那个时候，没有产生真正安全好用的电子支付手段。

所以，在72小时的网络生存测试中，有些人通过网络下单，网络支付的，确实是收到了东西。但也有一些，直到72小时活动结束，付出的电子货币都没能得到确认，自然也没收到到东西。

我睡了两觉，该吃的吃了该喝的喝了，就在房间里开始思考。

不思考也没事情可做啊，所以我就思考，不，发呆。不不，我是在等待。等待我买的东西，因为我买的好多东西都还没到。我已经尽量不买异地的东西了，肯定到不了的嘛，我只买本地的。床上用品牙刷牙膏一次性内裤那些我有了，吃的喝的我也不缺。我就看给我们服务的那几个网站还有些什么，是我能用的，又能送到的。女装，儿童玩具那些我不能买，摄像头盯着呢，电脑所有的操作被监控软件记录着呢。跟“生存”无关的东西我也不好意思买，因为测试里买的所有东西都是能带回家的，我不敢让人觉得我在占便宜（该死的要面子心理），所以有些挺贵的显卡内存条什么的，我没敢下单。那我能干什么呢，我买了几包巧克力送给我不同城市的朋友。反正我下单了，商家接单了，三天后还是三十天后送到，我真的无所谓。

当时的配送服务，即使是专为了活动特设的网站，也不定多长时间才能保证把你买的东西送到，因为都是由商家的员工甚至老板亲自送货自己开车或者打车的快点，骑自行车或者乘公交车的慢点，就这样。具体多久到？他也不清楚。都说是尽快，尽快是多快？基本上每个人都有下单后三五天才送到房间门口的，那时候我们已经走了。

72小时后，我们出来了。然后就是例行公事的采访，总结，各种该说不该说的，该做不该做的。作为一个有噱头的市场项目，72小时网络生存测试是成功的，它吸引了一大批人关注网络，关注如何用一种新的方式购物，去思考通过网络可以做些什么，这些人可能包括马云刘强东，也包括我。虽然活动过后没几个月，梦想家网站就倒闭了。

二十多年后，网络生存已经是一个极其容易的事情，或者说，网络已经成为生存的一部分。当时网络生存的三大难题：一、怎么找到要买的东西/网站；二、怎么有效安全地支付；三、商家怎么送货。——已经被中国人解决了，虽然还有可以改进的地方。但能够解决这三个问题的人，都是很牛逼的人。马云，开了一个全球最大的市场，买家卖家在这里交易，基本上所有合法的东西都能在这里买到。他还学习马斯克的PayPal，弄了个很安全的电子支付平台支付宝——所以马斯克最牛逼的事情是搞出了很好用的电子支付平台，特斯拉是后面的事。顺丰，三通一达这样的快递公司把点对点的物流几乎做到了极致。但实现这些，是在网络生存测试之后好几年的事。

在那之前，8848的王峻涛，本可以成为中国第一家也是最大的一家电子商务公司，但他把自己搞死了。2002年，我在北京见到他时，他已经失去了锐气。文人气息过浓的“老榕”再也没能翻身。

当当网，从卖书起家，一度成为最有商誉的购物网站。被市场搞死了。

易趣，理想主义者邵亦波的作品。过于理想话以至于商业角度上太脆弱，即使没有淘宝，也会被其他的购物网站干掉。

Eguo易国一小时，第一个打出同城购物一小时内送到的网站。在那个没有电动自行车，私家车不普及，大城市禁摩的年代，我不知道他是怎么做到从中心站点出发送货一小时到达的。过于理想化，细节不完善，死了。

必须要提一下，[http://cncard.com.cn](https://link.zhihu.com/?target=http%3A//cncard.com.cn)云网，我见到的唯一一个，可以独立谈下来工商银行电子支付渠道，并且将流程优化到极点的公司。不知道有没有人记得，在2002年前后，云网是最大的手机和游戏充值卡销售平台。当时几乎所有的购物网站电子支付都要通过一个统一的网银接口，再转到各家银行的接口，跳转三四个页面才能完成支付。但是云网，你只要点击购买，输入银行卡号，OK完成。这无限接近现在支付宝的支付体验，虽然他家只支持工行一家接口。我跟云网的老板聊了无数次，想租用他们的支付模块，都被拒绝了。我还记得老板夫妇两人的语气：我们知道你们都想要，但是给多少钱都不管用。你知道工行我们多难谈下来吗？我们可以承包你们的支付卡销售平台，但支付接口我们不卖。结果大家都知道了。

网易，新浪，搜狐，中华网，TOM.COM,各大门户网站的购物频道，首先从竞争中崛起，靠门户背后的人力和资源优势，一度与易趣和8848这样体量的独立购物平台打得不可开交，但随着互联网泡沫的涌起和破灭，门户网站迅速将“不挣钱”的电商部门砍掉了。2002年我试图重建中华网的电商平台，发现电商模块的源程序是来自国外，我辗转从网易得到了一套平台技术手册——没错，网易的电商平台系统购自中华网。那套打印版是我见过的最后一份国内电子商务平台技术资料，其中对电子商务的全部流程做了极度规范的设计。其后的淘宝，京东……都有那种原始的技术痕迹。那是原始的弓箭，所有火箭发动机的始祖。

从1999年到2025年，无数个理想化的电子商务先驱倒下了，他们有的是英雄，有的是傻子。淘宝和京东，拼多多这样的成功者还活得好好的，所以他们肯定不是笨蛋。

猜猜我是谁？
